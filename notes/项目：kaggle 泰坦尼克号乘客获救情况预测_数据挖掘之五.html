根据输出结果，可能会有一些想法：

  • 对于 Sex 属性，female 会极大提高最后获救的概率，而 male 会很大程度拉低这个概率；

  • 对于 Pclass 属性，1 等舱乘客最后获救的概率会上升，而乘客等级为 3 会极大地拉低这个概率。

  • 有 Cabin 值会很大程度拉升最后获救概率；

如果对上面的各种想法进行了实施，那么可能就需要来评判对应的想法是否能够有更好的效果；此时就需要用交叉验证(cross validation)来实现了。

2. 交叉验证

通常情况下这么做交叉验证：把 train.csv 分成两部分，一部分用于训练模型，另外一部分用来看预测算法的效果。

在这里使用 scikit-learn 的 cross_validation 来完成相关工作。

在进行交叉验证的时候，因为使用的是 train 数据，所以暂时先将 test 数据先注释掉。

简单交叉验证如下：

X_cross = X.copy()
y_cross = y.copy()

scores = cross_val_score(clf, X_cross, y_cross, cv=5)
print(scores)
print('Accuracy: %0.2f(+/- %0.2f)' % (scores.mean(), scores.std()*2))

输出结果如下：

[0.81564246 0.81564246 0.78651685 0.78651685 0.81355932]
Accuracy: 0.80(+/- 0.03)

共有 5 个数据。因为 cross_val_score(clf, X, y, cv=5) 中 cv 的值为 5。当 cv 参数是一个整数时， cross_val_score 默认使用 KFold 或 StratifiedKFold 策略，后者会在估计器派生自 ClassifierMixin 时使用；此时 cv 的值表示的是 KFold 或 StratifiedKFold 中所分的“折数”；所以输出的结果也是有 5 个。（更多内容详见相关资料）

接下来可以利用交叉验证将预测结果里面bad case部分显示出来，看看通过人眼审核能不能看出更多的信息(可以与前面的相关系数的结果结合起来看看)。

但是接下来可以根据以下的一些思路来对模型进行优化：

  • Age 属性不使用现在的拟合方式，而是根据名称中的 “Mr”、“Mrs”、“Miss” 等的平均值进行填充；

  • Age 不做成一个连续值属性，而是使用一个步长进行离散化，变成离散的类目 feature；

  • Cabin 再细化一些，对于有记录的 Cabin 属性，我们将其分为前面的字母部分(我猜是位置和船层之类的信息) 和 后面的数字部分(应该是房间号，有意思的事情是，如果你仔细看看原始数据，你会发现，这个值大的情况下，似乎获救的可能性高一些)；

  • Pclass 和 Sex 俩太重要了，我们试着用它们去组出一个组合属性来试试，这也是另外一种程度的细化；

  • 单加一个 Child 字段，Age<=12 的，设为 1，其余为 0 (可以去看看数据，确实 Age 较小的优先程度更高)；

  • 如果名字里面有 “Mrs”，而 Parch>1 的，我们猜测她可能是一个母亲，应该获救的概率也会提高，因此可以多加一个 Mother 字段，此种情况下设为 1，其余情况下设为 0；

  • 登船港口可以考虑先去掉试试(Q 和 C 本来就没权重，S 有点奇怪)；

  • 把堂兄弟/兄妹 和 Parch 还有自己 个数加在一起组成一个Family_size特征(考虑到大家族可能对最后的结果有影响)；

  • Name是一个我们一直没有触碰的属性，我们可以做一些简单的处理，比如说男性中带某些字眼的(‘Capt’, ‘Don’, ‘Major’, ‘Sir’)可以统一到一个 Title，女性也一样。

还有更多的内容可以被挖掘。

3. 学习曲线

如果不断地做 feature engineering，产生的特征越来越多，那么训练出来的模型可能就会产生 过拟合 问题。除此之外，模型也可能会出现 欠拟合 问题。

对于过拟合和欠拟合问题，处理的方法是不一样的。

可以通过学习曲线(learning curve)来判断模型是出现了过拟合还是欠拟合。

# 绘制学习曲线
X_curve = X.copy()
y_curve = y.copy()

train_sizes, train_scores, test_scores = learning_curve(
    clf, X_curve, y_curve, cv=6, train_sizes=np.linspace(0.1, 1.0, 10))

train_scores_mean = np.mean(train_scores, axis=1)
train_scores_std = np.std(train_scores, axis=1)
test_scores_mean = np.mean(test_scores, axis=1)
test_scores_std = np.std(test_scores, axis=1)

fig1 = plt.figure(num=1, figsize=(24, 18))
ax1 = fig1.add_subplot(1, 1, 1)

plt.plot(train_sizes,
         1 - train_scores_mean,
         label='train data',
         color='r',
         marker='o')
plt.plot(train_sizes,
         1 - test_scores_mean,
         label='test data',
         color='g',
         marker='o')
plt.legend()
plt.grid()
plt.xlabel('Data Number')
plt.ylabel('Error')
plt.title('Learning Curves')

plt.show()

除此之外，我们可以再做些 feature engineering 的工作，添加一些新产出的特征或者组合特征到模型中。

模型融合：

模型融合(model ensemble)大概是：有一堆在 同一份数据集 上训练得到的分类器(比如logistic regression，SVM，KNN，random forest，神经网络)，那我们让他们都 分别 去做判定，然后对结果做投票统计，取票数最多的结果为最后结果。

如果不使用全部的训练集，每次取训练集的一个subset做训练，这样的话，虽然用的是同一个机器学习算法，但是得到的模型却是不一样的；同时，因为我们没有任何一份子数据集是全的，因此即使出现过拟合，也是在子训练集上出现过拟合，而不是全体数据上，这样做一个融合，可能对最后的结果有一定的帮助。对，这就是常用的Bagging。（更确切的说，这是另一种思路和方向，是将弱分类器通过学习算法集成起来变为很强的分类器，称为机器学习元算法。Bagging 是其中的一种）

因为针对当前的逻辑回归模型，我们目前还有很多想法想要去尝试，所以这里先把模型融合放一放，到了后面的时候再实现该想法。
